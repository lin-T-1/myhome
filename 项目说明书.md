# 智能家居课程项目说明书

## 一、项目概述

本项目基于 **STM32F103C8T6**（Blue Pill 开发板），使用 **STM32 HAL 库** 与 **Keil MDK-ARM** 开发，实现一套简易智能家居演示系统。系统通过光敏、热敏等传感器采集环境信息，自动控制照明与报警，并在 **OLED 显示屏** 上实时展示状态。

**主控芯片：** STM32F103C8T6  
**开发环境：** STM32CubeMX + Keil MDK-ARM V5  
**工程名称：** Smarthome_Course  

---

## 二、实现功能

### 2.1 自动照明

| 功能 | 说明 |
|------|------|
| 检测环境光 | 光敏电阻模块（5516）通过 ADC 读取光照强度 |
| 自动开灯 | 环境变暗（遮光）时，**PA5** 输出高电平，点亮外接 LED |
| 自动关灯 | 环境变亮时自动关闭 LED |
| 滞回控制 | 开灯、关灯使用不同阈值，避免在临界亮度附近反复开关 |

### 2.2 温度报警

| 功能 | 说明 |
|------|------|
| 温度采集 | NTC 热敏模块通过 ADC 读取原始值（OLED 显示为 **Tp**） |
| 高温报警 | 当 **Tp 低于 1800**（温度升高、ADC 值降低）时，蜂鸣器持续鸣叫 |
| 恢复静音 | 温度恢复正常后自动停止 |
| 上电保护 | 上电后约 **800 ms** 内不报警，避免误触发 |

### 2.3 OLED 状态显示

128×64 OLED（I2C 接口，软件模拟）显示以下内容：

```
ADC: xxxx        ← 光敏模块 ADC 原始值
Tp  xxxx         ← 热敏模块 ADC 原始值
Lamp: ON/OFF     ← 外接照明 LED 状态
Amb: BRIGHT/DARK ← 环境明暗（BRIGHT=有光，DARK=暗）
```

> 说明：数据变化超过一定幅度时才刷新屏幕，减少闪烁。

### 2.4 系统指示

- 上电后 **PC13 板载 LED** 闪烁 3 次，表示程序已正常运行。
- 启动界面显示 `SMART HOME` / `Ready...`。

### 2.5 预留模块（已接线/有驱动，当前主逻辑未启用）

工程中包含 **反射式红外传感器** 驱动（`ir_sensor.c`），GPIO 已配置，可按需扩展：

| 模块 | 引脚 | 原设计用途 |
|------|------|------------|
| 人体反射红外 | PA3 | 人体检测 |
| 门禁反射红外 | PA4 | 遮挡检测 |

当前版本主控逻辑由 **温度** 触发蜂鸣器，红外模块未参与 `smart_home.c` 主循环。

## 四、引脚分配表

| STM32 引脚 | 外设/模块 | 方向 | 说明 |
|------------|-----------|------|------|
| **PA1** | 光敏模块 AO | 模拟输入 | ADC1 通道 1，读光照（接 AO，勿接 DO） |
| **PA2** | 热敏模块 AO | 模拟输入 | ADC1 通道 2，读温度 |
| **PA3** | 人体反射红外 DO | 输入（上拉） | 无遮挡=高，有遮挡=低（预留） |
| **PA4** | 门禁反射红外 DO | 输入（上拉） | 无遮挡=高，有遮挡=低（预留） |
| **PA5** | 外接 LED | 输出 | 高电平点亮 |
| **PB1** | 有源蜂鸣器 | 输出 | 低电平响，高电平停 |
| **PB8** | OLED SCL | 开漏输出 | 软件 I2C 时钟 |
| **PB9** | OLED SDA | 开漏输出 | 软件 I2C 数据 |
| **PC13** | 板载 LED | 输出 | 低电平点亮 |

## 五、接线说明

### 5.1 光敏模块（5516）

```
VCC → 3.3V
GND → GND
AO  → PA1
DO  → 不接（DO 指示灯由模块电位器控制，与单片机无关）
```

### 5.2 热敏/NTC 模块

```
VCC → 3.3V
GND → GND
AO  → PA2
```

### 5.3 OLED

```
VCC → 3.3V
GND → GND
SCL → PB8
SDA → PB9
```

### 5.4 有源蜂鸣器

```
正极 → PB1（或通过三极管驱动）
负极 → GND
```
> 程序配置：低电平响，高电平停。

### 5.5 外接 LED（照明）

```
PA5 → 限流电阻（330Ω~1kΩ）→ LED 正极
LED 负极 → GND
```

### 5.6 反射式红外（可选）

```
VCC → 3.3V 或 5V（视模块而定）
GND → GND
DO  → PA3（人体）或 PA4（门禁）
```

## 六、软件模块结构

```
Core/Src/
├── main.c           主程序入口，系统初始化与主循环
├── smart_home.c     智能家居核心逻辑（传感、控制、显示调度）
├── adc_sensor.c     ADC 光敏/热敏采集与滤波
├── led.c            外接 LED 与板载 LED 驱动
├── buzzer.c         蜂鸣器驱动
├── ir_sensor.c      反射红外驱动（预留）
├── oled.c           OLED 显示驱动（软件 I2C）
└── gpio.c           GPIO 初始化
```

**主循环逻辑：**

- 每 **20 ms** 执行控制逻辑。
- 每 **200 ms** 读取传感器并刷新 OLED。

## 七、关键参数（可在 smart_home.c 中修改）

| 参数 | 默认值 | 含义 |
|------|--------|------|
| `LIGHT_DARK_RAW_ON` | 2000 | 遮光超过此 ADC 值开灯 |
| `LIGHT_DARK_RAW_OFF` | 2150 | 亮灯后 ADC 低于此值关灯 |
| `TEMP_HIGH_RAW_THRESH` | 1800 | Tp 低于此值触发高温报警 |
| `BUZZER_ARM_MS` | 800 | 上电后报警延时（ms） |

## 八、编译与下载

1. 使用 **Keil MDK-ARM** 打开 `MDK-ARM/Smarthome_Course.uvprojx`。
2. 编译工程（F7）。
3. 通过 ST-Link 或 USB 串口下载器烧录到 STM32F103C8T6。
4. 上电后观察 PC13 闪烁 3 次及 OLED 显示，即表示程序运行正常。

## 十、注意事项

1. **PA1、PA2 必须接模块 AO（模拟输出）**，接 DO 会导致读数异常甚至芯片发热。
2. **PB6、PB7 未用于 OLED**，OLED 使用 PB8/PB9 软件 I2C。
3. 光敏、热敏阈值需根据实际模块与环境在 `smart_home.c` 中微调。
4. 反射红外模块已预留引脚与驱动，扩展时可重新接入 `smart_home.c` 主逻辑。
